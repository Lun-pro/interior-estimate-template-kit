  # 공개용 샘플 데이터 작성 가이드

이 문서는 오픈소스 문서, 블로그, SNS, 앱 소개 화면, 테스트 화면에 샘플 데이터를 공개할 때 참고할 수 있는 개인정보 보호 가이드입니다.

실제 고객 정보, 실제 현장 정보, 실제 업체 정보, 실제 계약 정보를 공개하지 않기 위한 기본 기준을 정리합니다.

## 1. 목적

견적서, 일정, 지출, 고객 정보와 관련된 예시를 공개할 때는 실제 개인정보가 포함되지 않도록 주의해야 합니다.

공개용 샘플 데이터는 실제 데이터를 그대로 사용하는 것이 아니라, 외부에 공개되어도 문제가 없는 가상의 정보로 작성해야 합니다.

## 2. 공개하면 안 되는 정보

다음 정보는 오픈소스 문서, 블로그, SNS, 샘플 이미지에 포함하지 않는 것이 좋습니다.

* 실제 고객명
* 실제 전화번호
* 실제 현장 주소
* 실제 동 / 호수
* 실제 이메일
* 실제 사업자등록번호
* 실제 계좌번호
* 실제 예금주
* 실제 계약 금액
* 실제 견적 금액
* 실제 자재 단가
* 실제 거래처 정보
* 실제 작업자 이름
* 실제 분쟁 사례
* 실제 내부 정산 기준

## 3. 공개용 샘플에 사용할 수 있는 정보

공개용 샘플에는 가상의 정보를 사용하는 것이 안전합니다.

| 항목      | 공개용 예시              |
| ------- | ------------------- |
| 고객명     | 홍길동 고객님             |
| 연락처     | 010-0000-0000       |
| 현장명     | 아파트 인테리어 샘플 현장      |
| 현장 주소   | ○○시 ○○구 ○○아파트       |
| 업체명     | 샘플 인테리어             |
| 사업자등록번호 | 000-00-00000        |
| 계좌번호    | 테스트은행 000-0000-0000 |
| 견적 금액   | 000,000원            |

위 정보는 실제 정보가 아니라 문서 구조를 설명하기 위한 더미 데이터입니다.

## 4. 주소 작성 기준

공개용 샘플에서는 실제 상세 주소를 쓰지 않는 것이 좋습니다.

사용 가능한 예시:

* ○○시 ○○구 ○○아파트
* 경남 ○○시 샘플 현장
* 서울 ○○구 아파트 샘플
* 현장 주소 비공개

사용하지 않는 것이 좋은 예시:

* 실제 도로명 주소
* 실제 아파트명과 동 / 호수
* 실제 고객 거주지
* 실제 현장 출입 정보

## 5. 연락처 작성 기준

공개용 샘플에서는 실제 전화번호를 사용하지 않습니다.

사용 가능한 예시:

* 010-0000-0000
* 000-0000-0000
* 연락처 비공개

사용하지 않는 것이 좋은 예시:

* 실제 고객 전화번호
* 실제 업체 대표 전화번호
* 실제 작업자 개인 연락처

## 6. 금액 작성 기준

공개용 샘플에서는 실제 계약 금액이나 실제 단가를 그대로 사용하지 않는 것이 좋습니다.

사용 가능한 예시:

* 000,000원
* 샘플 금액
* 협의 후 결정
* 금액 비공개

사용하지 않는 것이 좋은 예시:

* 실제 견적 금액
* 실제 계약 금액
* 실제 자재 단가
* 실제 외주 비용
* 실제 수익률

## 7. 이미지 공개 전 체크리스트

블로그, SNS, GitHub, 앱 소개 이미지에 화면 캡처를 올리기 전에는 아래 항목을 확인하는 것이 좋습니다.

* 실제 고객명이 보이지 않는가?
* 실제 전화번호가 보이지 않는가?
* 실제 주소가 보이지 않는가?
* 실제 동 / 호수가 보이지 않는가?
* 실제 계좌번호가 보이지 않는가?
* 실제 사업자등록번호가 보이지 않는가?
* 실제 계약 금액이 보이지 않는가?
* 실제 Firebase, Play Console, 계정 정보가 보이지 않는가?
* 앱 내부 테스트 계정 정보가 보이지 않는가?

## 8. 오픈소스 문서 작성 시 주의사항

오픈소스 문서에는 누구나 볼 수 있는 정보만 포함해야 합니다.

특정 앱이나 업체의 내부 운영 기준을 그대로 공개하지 않고, 일반적인 예시와 작성 방향만 제공하는 것이 안전합니다.

공개 문서에 적합한 내용:

* 일반적인 항목 설명
* 개인정보 보호 기준
* 더미 데이터 작성 방법
* 공개 전 확인 체크리스트

공개 문서에 적합하지 않은 내용:

* 실제 고객 데이터
* 실제 업체 내부 기준
* 실제 계약 조건
* 실제 원가 구조
* 실제 운영 전략

## Summary

This guide provides a simplified public reference for creating privacy-safe sample data.

It is intended to help small interior contractors and freelancers share example documents, screenshots, and templates without exposing real customer data, field addresses, payment details, or private business information.
